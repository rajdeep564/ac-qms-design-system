This matrix is enforced at three layers: API middleware (role guards), database (no UPDATE/DELETE permissions on audit_logs), and UI (role-conditional rendering). The no-self-approval rule applies in addition to this matrix — a user cannot perform two consecutive workflow stages on the same document. See ICH Q7 Section 6 and 21 CFR Part 11 §11.10 for regulatory basis.
