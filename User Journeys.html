Step 1 Rajesh Kumar · Super Admin

Before any work can happen, Rajesh creates the user accounts and approves the product master for Glycine IP. The master is the single source of truth — nothing else in the system can exist without it being signed off first.

Created accounts for QC and QA team members
Approved Product Master FG00038 (Glycine IP, Revision 03)

Step 2 Priya Mehta · QC Manager

Production has just delivered a fresh batch of Glycine IP, batch number GCN/010526. Priya opens the QC dashboard, creates the batch entry, assigns it to her analyst Kavya, and the system automatically generates the ARN: 2026FG03. From here, the four-document chain begins — SPEC, then MOA, then AWS, then COA.

Created batch GCN/010526 in the system
Assigned to Kavya Patel (QC Executive)
ARN auto-generated: 2026FG03
See Priya's dashboard
Step 3 Kavya Patel · QC Executive

Kavya opens her assigned batch and starts the SPEC — the document that defines exactly what tests will run on this batch and what results are acceptable. The system guides her through the tests one by one, locks the criteria that come from the Master, and lets her add any customer-specific overrides. When she's confident, she submits it — and the system asks her to re-enter her password as a legal signature.

Reviewed all 11 mandatory tests for Glycine IP
Added customer-specific notes
Submitted SPEC/FG00038/01 with password and location captured
Walk through SPEC creation
Step 4 Priya Mehta · QC Manager (again)

Priya gets notified that the SPEC needs her approval. She opens it in read-only mode, checks every test and criterion, and either approves or sends it back with a comment. This is the first of three sign-offs the document will need before it's official.

Reviewed Kavya's SPEC submission
Approved with password and location
See the approval screen
Step 5 Sanjay Reddy · QA Manager

Sanjay is the final authority. He reviews the QC-approved SPEC and signs it. The moment he signs, the MOA — the Method of Analysis document — is auto-created using procedures defined in the Master. This is how AC-QMS keeps everything connected: signing one document triggers the next one automatically. After SPEC comes MOA, then AWS (the actual lab results), then finally COA — the Certificate of Analysis.

Signed SPEC — MOA auto-created
Signed MOA — AWS unlocked for Kavya
Signed AWS — COA auto-generated
See Sanjay's QA dashboard
Step 6 · Batch Released Sanjay Reddy · QA Manager

Once all test results are signed off, the COA auto-generates from the lab data — no manual drafting, no typos. Sanjay reviews it one final time and clicks "Sign & Issue." That single action releases the batch: Production and Stores are notified, the batch status changes to RELEASED, and Glycine IP batch GCN/010526 is now ready to ship. The complete audit trail — every signature, every acceptance, every location — is permanently preserved.

Signed and issued — batch released
